## 🚀 How It Works

```
┌─────────────────────────────────────────┐
│ Client Request Arrives at /auth/login   │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ Get Client ID (User ID or IP Address)   │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ Check Redis for requests in last 15min  │
└─────────────────────────────────────────┘
              ↓
        ┌─────────────┬────────────────┐
        ↓             ↓                 ↓
    ┌────────┐  ┌──────────┐
    │4 Reqs  │  │5 Reqs    │
    │<Limit  │  │At Limit  │
    └────────┘  └──────────┘
        ↓             ↓
    ALLOW ✅      BLOCK ❌
    (Pass)        (429 Error)
    
Response Headers:
├─ X-RateLimit-Limit: 5
├─ X-RateLimit-Remaining: 1
├─ X-RateLimit-Reset: (unix timestamp)
└─ (if blocked) Retry-After: 300 seconds
```

## 💡 Smart Features

### 1. User-Aware
```
Authenticated User    = 2.5x limit boost
├─ Login limit: 5 → (not boosted for sensitive endpoints)
├─ File limit: 100 → 250/15min
└─ Admin limit: 50 → 125/15min

Unauthenticated User  = Normal limit
├─ Public limit: 200/15min
└─ API limit: 100/15min
```

### 2. Method-Aware
```
GET    = Normal (read-only)
POST   = 0.8x (data creation)
PUT    = 0.8x (data modification)
DELETE = 0.5x (destructive - strictest!)
PATCH  = 0.8x
```

### 3. Endpoint-Aware
```
/auth/*     → Strictest
/admin/*    → Strict
/file/*     → Moderate
/public/*   → Lenient
/api/*      → Standard
```

### 4. Error Handling
```
Redis unavailable? → Allow request (fail-safe)
Invalid config?    → Use defaults
Rate limit check error? → Allow request + log
```

## 🛠️ Customization Examples

### Change Default Limit
```javascript
// In backend/config/rateLimitConfig.js
defaults: {
    max: 150,  // Changed from 100
    windowMs: 10 * 60 * 1000,
}
```

### Change Auth Limit
```javascript
auth: {
    max: 10,  // More attempts (was 5)
    windowMs: 30 * 60 * 1000,  // 30 minutes
}
```

### Boost User Multiplier
```javascript
authenticated: {
    multiplier: 3  // Was 2.5x
}
```

### Add Custom Rate Limit to New Route
```javascript
import { customRateLimiter } from '../middleware/rateLimiter.js'

router.post('/api/expensive-op',
    customRateLimiter({ max: 10, windowMs: 60*60*1000 }),
    handler
)
```
